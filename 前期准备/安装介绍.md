# 语言介绍

## 诞生背景

Google 公司允许每个员工拥有 20% 的自由时间来开发一些实验性的项目。随着 C++ 特性的不断增多，Google 公司旗下员工 Rob Pike 觉得 C++ 应该得到一些简化，因为此时的 C++ 已经足够复杂了。

基于此种原因，Rob Pike 内心创造一个新语言的想法愈来愈强烈，随后 Rob Pike 开始联合一些大神利用这 20% 的空闲时间开发这一新的语言。

终于，在 2006 年 1 月 2 日，Golang 正式诞生了，并在 2009 年 11 月进行开源，随后在 2012 年发布第一款稳定版本。

可以说，Golang 是一款非常年轻的现代化语言，而 Rob Pike 也被人们亲切地称为 Go 语言之父。

## 语言思想

- Less can be more（少即是多）
- 大道至简，小而蕴真
- 让事情变得复杂很容易，让事情变得简单才难

## 语言特点



- 简洁的语法，令其编码效率可比肩 Python
- 高效的性能，令其执行效率可比肩 C 语言
- 跨平台编译，令开发环境的要求越来越低
- 自带 GC 机制，令开发人员无需过分关注底层



# 安装Go

## Linux安装

以 Linux 平台为例，打开 Golang 官网：[golang.google.cn/](https://golang.google.cn/)

<img src="https://images-1302522496.cos.ap-nanjing.myqcloud.com/img/202112222253559.png" alt="image-20211222225346450" style="zoom:60%;" />

点击 Download 后，选择需要下载的版本：

<img src="https://images-1302522496.cos.ap-nanjing.myqcloud.com/img/202112231435995.png" alt="image-20211223143501873" style="zoom:73%;" />

这里我选择的是 Linux x86 64 位的版本，直接下载即可：

```
$ wget https://go.dev/dl/go1.17.5.linux-amd64.tar.gz
```

然后解压该软件包，我将它解压到了 /usr/local 目录下：

```
$ sudo tar xvf go1.17.5.linux-amd64.tar.gz -C /usr/local
```

接下来到 ~/.zshrc 中配置环境变量，加入下面 4 行配置：

```
$ vim ~/.zshrc

export GOROOT=/usr/local/go
export GO111MODULE=on
export GOPROXY="https://proxy.golang.org,direct"
export PATH=$PATH:$GOROOT/bin

// GOROOT：指定 Go 的安装目录
// GO111MODULE：开启 go mod 模块管理
// GOPROXY：设置 Go 代理
```

刷新该配置文件：

```
$ source ~/.zshrc
```

打印 Golang 的版本信息，如果出现下面的字样则代表配置完成：

```
$ go version   
go version go1.17.5 linux/amd64
```

其他平台可参照官方文档进行安装：[golang.google.cn/doc/install](https://golang.google.cn/doc/install)



## 目录结构

查看 Golang 的安装目录结构，各个子目录的功能用途如下所示：

```
$ tree /usr/local/go -L 1

$ tree /usr/local/go -d -L 1
/usr/local/go
├── api   // 存放每个版本的 API 变更差异
├── bin   // 存放编译器 (go)、文档工具 (godoc)、格式化工具 (gofmt)
├── doc   // 存放英文版的 go 文档
├── lib   // 存放引用的一些库文件
├── misc  // 存放杂项用途文件，如 Android 平台的编译、git 的提交钩子等
├── pkg   // 存放编译好的一些内置库文件
├── src   // 存放标准库的源码文件
└── test  // 存放测试用例
```

# 快速上手

## hello world

现在让我们编写第一个 Go 程序，首先新建一个 Go 项目：

```
$ mkdir -p ~/project/go/demo
$ cd ~/project/go/demo
$ go mod init example/hello  // 在当前目录下初始化一个 module
$ vim ./hello.go
```

写入以下代码并退出保存：

```
package main  // 声明这是一个 main 包

import "fmt"  // 导入 fmt 模块

func main() {
	fmt.Printf("Hello World \n")
}
```

然后在终端输入以下命令：

```
$ go run ./hello.go 
Hello World 
```

# 编译相关

## go build

go build 命令可以将 Golang 的源代码编译成可执行文件。

如下所示，尝试对示例项目 example/hello 进行编译，使用 -o 指定输出后的可执行文件名：

```
$ go build -o hello
```

然后直接执行该文件：

```
$ ./hello
Hello, World
```

如果不指定 -o 参数，Golang 则会自动寻找 go.mod 文件，并将生成的可执行文件依照 go.mod 文件中的 module 配置项进行命名。

这个 go.mod 文件是在执行 go mod init 命令时生成的文件，文件内容如下，其中 module 的命名是在执行 go mod init 命令时指定的：

```
module example/hello

go 1.17
```



## go run

go run 命令可以直接编译并执行 Golang 的源码文件，但是编译后的可执行文件不会在当前目录下存放而是存放在临时目录中：

```
$ go run ./hello.go    
Hello World
```





## 交叉编译

默认的 go build 命令编译出的可执行文件仅针对当前平台，如果想进行跨平台编译，则需要指定目标操作系统的平台和处理器架构，以 Linux 下编译 Windows 中可执行文件为例：

```
$ CGO_ENABLED=0   // 禁用 CGO 后才能支持跨平台编译
$ GOOS=windows    // 目标平台
$ GOARCH=amd64    // 目标处理器架构
$ go build        // 进行编译，不指定文件会自动编译当前项目
```

此外，需要额外注意的 2 个点：

- 对于在 Mac 或者 Linux 平台下运行交叉编译后生成的可执行文件而言，需要使用 chmod 命令为其增加可执行权限
- 对于在 Windows 平台下运行交叉编译后生成的可执行文件而言，我们需要到 cmd 中进行，powershell 中是不支持的

Mac 下编译 Linux 中的 64 位可执行程序：

```
CGO_ENABLED=0 
GOOS=linux 
GOARCH=amd64 
go build
```

Mac 下编译 Windows 中的 64 位可执行程序：

```
CGO_ENABLED=0
GOOS=windows
GOARCH=amd64
go build
```

Linux 下编译 Mac 中的 64 位可执行程序：

```
CGO_ENABLED=0
GOOS=darwin
GOARCH=amd64
go build
```

Linux 下编译 Windows 中的 64 位可执行程序：

```
CGO_ENABLED=0
GOOS=windows
GOARCH=amd64
go build
```

Windows 下编译 Mac 中的 64 位可执行程序：

```
SET CGO_ENABLED=0
SET GOOS=darwin
SET GOARCH=amd64
go build
```

Windows 下编译 Linux 中的 64 位可执行程序：

```
SET CGO_ENABLED=0
SET GOOS=linux
SET GOARCH=amd64
go build
```
