# 基础结构

## main

一个 Go 程序的基础结构如下：

```
package main

import "fmt"

func main() {
	fmt.Printf("Golang ... \n")
}
```

释义如下：

- package main：声明这是一个 main 包，一个 Go 项目必须拥有 1 个 main 包，这样才能编译出 1 个可执行文件
- func main：声明整个项目的入口函数，一个 main 包中必须具有 main 函数，它将作为该项目的入口函数存在，且会在执行 Go 程序时自动运行
- import "fmt"：这是一段导入其他包的语句，对于 Go 的基础结构来说，它并不是必须的



对于目前的单文件 Go 程序而言，所有的代码都应该书写在 main 入口函数中。

入口函数外的区域只能书写一些全局变量、常量、函数名以及 import 导入语句。



# 注释方式

## 单行注释

在 Golang 中，单行注释为 // ：

```
package main

import "fmt"

func main() {
	// this is line comment
	fmt.Printf("Golang ... \n")
}
```



## 多行注释

在 Golang 中，多行注释为 /\* \*/：

```
package main

import "fmt"

func main() {
	/*
	   this is block comment
	*/
	fmt.Printf("Golang ... \n")
}
```



# 命名方式

## 命名风格

在 Golang 中，标识符（所有资源的名称）的命名必须是由字母、数字、下划线组成，且不能由数字开头。

Golang 推荐使用驼峰式的命名风格，并且针对小驼峰或者大驼峰都规定了不同的语义：

- 大驼峰：表明该标识符资源是对外公布的
- 小驼峰：表明该标识符资源是对外隐藏的

具体的作用在后面会有详细的介绍，这里不再进行讲解。



## 关键字

关键字是指编程语言中已经定义好的且应用于语法中的标识符，不要使用关键字作为自定义的标识符。

Golang 中拥有 25 个关键字：

```

case         defer        go           map          struct
chan         else         goto         package      switch
const        fallthrough  if           range        type
continue     for          import       return       var
```



## 保留字

保留字是指编程语言中已经定义好的且具有特殊意义的标识符，不要使用保留字作为自定义的标识符。

Golang 中拥有 37 个保留字：

# 声明方式

## 局部声明

在 main 函数内声明的标识符均属于局部声明的标识符。

通常情况下，局部声明的标识符必须使用，否则会抛出异常。这项严苛的规则避免了编译后可执行文件体积过大的问题。

在下面这个例子中我们局部声明了一个变量 userName，但是没有对其进行使用，所以在 go run 时抛出了 userName declared but not used 的异常：

```
package main

import "fmt"

func main() {
	var userName string = "Jack"
	fmt.Printf("Golang ... \n")
}
```



## 全局声明

在 mian 函数外声明的标识符均属于全局声明的标识符。

全局声明的标识符可以在声明后不使用，这样做并不会抛出异常：

```
package main

import "fmt"

var userName string = "Jack"

func main() {
	fmt.Printf("Golang ... \n")
}
```



# 内置功能

## 内置函数

不同于其他语言，Golang 中自带的内置函数很少，绝大部分功能函数都来自于内置库。

举个例子，对于字符串的 Split 方法而言，Golang 不会将这个方法定义在 String 的结构体中，而是将其存放在了 strings 的内置包里。

以下是 Golang 中的内置函数一览：

```
append          -- 用来追加元素到数组、slice 中,返回修改后的数组、slice
close           -- 主要用来关闭 channel
delete          -- 从 map 中删除 key 对应的 value
panic           -- 停止常规的 goroutine（panic 和 recover：用来做错误处理）
recover         -- 允许程序定义 goroutine 的 panic 动作
real            -- 返回 complex 的实部（complex、real imag：用于创建和操作复数）
imag            -- 返回 complex 的虚部
make            -- 用来分配内存，返回 Type 本身(只能应用于 slice, map, channel)
new             -- 用来分配内存，主要用来分配值类型，比如 int、struct。返回指向 Type 的指针
cap             -- capacity 是容量的意思，用于返回某个类型的最大容量（只能用于切片和 map）
copy            -- 用于复制和连接 slice，返回复制的数目
len             -- 来求长度，比如 string、array、slice、map、channel，返回长度
print、println   -- 底层打印函数，在部署环境中建议使用 fmt 包
```





## 内置接口

Golang 中内置了一个接口 error：

```
type error interface {  // 只要实现了 Error 函数且返回值为 String 的都实现了 err 接口
    Error()    String
}
```
