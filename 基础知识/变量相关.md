# 标准声明

变量用于存储一些在程序运行中可能会发生改变的数据。

Golang 中变量的标准声明必须使用 var 关键字开头：

```
var 变量名 变量类型  // 变量声明
var 变量名 = 变量值  // 变量赋值
```

代码示例，我们声明了一个 string 类型的变量，名称为 userName，并将字符串 Jack 赋值给了 userName ：

```
package main

import "fmt"

func main() {
	var userName string
	userName = "Jack"
	fmt.Printf("%#v \n", userName)
}

// "Jack"
```

在 Golang 中，变量必须先定义、后使用，否则将会抛出异常：

```
package main

import "fmt"

func main() {
	fmt.Printf("%#v \n", userName)
	var userName string
	userName = "Jack"
}

// undefined: userName
```

在 Golang 中，同一个作用域下的变量不允许被重复定义，否则将会抛出异常：

```
package main

import "fmt"

func main() {
	var userName string
	var userName string
	fmt.Printf("%#v \n", userName)
}

// userName redeclared in this block
```



# 定值声明

我们可以在对变量进行声明的时候一同对其进行赋值，这是 Golang 推荐的做法：

```
var 变量名 变量类型 = 变量值
```

代码示例：

```
package main

import "fmt"

func main() {
	var userName string = "Jack"
	fmt.Printf("%#v \n", userName)
}

// "Jack"
```



# 类型推导

Golang 自带类型推导机制，我们可以在使用定值声明时不指定变量的类型，转而由 Go 编译器自动推导出该变量的类型：

```
var 变量名 = 变量值
```

代码示例：

```
package main

import "fmt"

func main() {
	var userName = "Jack"
	fmt.Printf("%T \n", userName)
	fmt.Printf("%#v \n", userName)
}

// string
// "Jack"
```



# 批量声明

Golang 中支持批量声明多个变量：

```
var (
	变量名 变量类型 = 变量值
	变量名 变量类型 = 变量值
	变量名 变量类型 = 变量值
)
```

也可以在批量声明时使用类型推导：

```
var (
	变量名 = 变量值
	变量名 = 变量值
	变量名 = 变量值
)
```

代码示例：

```
package main

import "fmt"

func main() {
	var (
		userName   string = "Jack"
		userAge    int    = 18
		userGender bool   = true
	)
	fmt.Printf("user name : %#v\nuser age : %#v\nuser gender : %#v\n", userName, userAge, userGender)
}

// user name : "Jack"
// user age : 18
// user gender : true
```



# 短变量

类型推导使我们可以省略变量类型的书写。

如果想更进一步省略 var 关键字的书写，则可以使用短变量声明。

需要注意的是短变量声明只能在局部作用域中使用，否则会抛出异常。

语法格式如下：

```
变量名 := 变量值
```

代码示例：

```
package main

import "fmt"

func main() {
	userName := "Jack"
	fmt.Printf("%T \n", userName)
	fmt.Printf("%#v \n", userName)
}

// string
// "Jack"
```



# 匿名变量

一些函数的返回值可能有多个，但并不是每一个都是我们所需要的。

此时可以使用匿名变量 _ 对不需要的返回值进行解构接收，匿名变量即使在局部作用域中定义后不进行使用，也不会抛出异常。

代码示例：

```
package main

import "fmt"

func isOdd(number int) (bool, string) {
	if number%2 != 0 {
		return true, fmt.Sprintf("%#v is odd number", number)
	}
	return false, fmt.Sprintf("%#v is not odd number", number)
}

func main() {
	tag, _ := isOdd(1)
	fmt.Printf("%#v \n", tag)
}

// true
```



# 类型零值

所有类型都具有 1 个零值，当变量进行标准声明后且未赋值前，该变量将存储该类型的零值：

- 整形和浮点型变量的零值均为 0
- 布尔类型变量的零值为 false
- 字符串变量的零值为空字符串
- 引用类型变量的零值均为 nil
- 接口和函数变量的零值均为 nil



# 个人风格

上面例举了很多关于变量的定义方式，那么这么多的方式该如何选择呢？

下面是我的 Golang 编码风格：

```
package main

import "fmt"

func get_gender(gender bool) string {
	if gender {
		return "male"
	}
	return "female"
}

func main() {
	// 对于普通的变量来说，使用定值声明
	var gender bool = true
	var level uint8 = 31

	// 对于数组、切片、映射等容器类型的变量使用类型推导
	var hobby = [3]string{"basketball", "football", "volleyball"}

	// 在获取函数返回结果、循环遍历时使用短变量
	gender_string := get_gender(gender)
	for _, element := range [3]string{"a", "a", "c"} {
		fmt.Printf("element : %#v \n", element)
	}

	fmt.Printf("gender : %#v \n", gender_string)
	fmt.Printf("level  : %d \n", level)
	fmt.Printf("hobby  : %#v \n", hobby)
}

// element : "a"
// element : "a"
// element : "c"
// gender : "male"
// level  : 31
// hobby  : [3]string{"basketball", "football", "volleyball"}
```
